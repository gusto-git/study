웹 개발, 웹 퍼블리싱(웹디자인과 웹 프로그래밍의 중간단계) 지금은 [서비스]가 주도하는 시대 >> 유튜브, 넷플릭스, 아마존, 배달의민족, 쿠팡, 직방 ... 본격적으로 서비스가 주도하는 시대 -> 서비스를 통해 새로운 가치 창출이 이루어짐 >> 이 서비스들은 웹에서 부터 시작한다. WEB : 연결고리 Windows, 데스크탑과 랩탑, 태블릿과 스마트폰, 안드로이드와 아이폰 등 과 모두를 이어주는 매개체 = WEB 예전에는 하나의 서비스를 만들기 위해 수십개 어플리케이션을 만들어야 했던 시절이 있음 WORA(Write Once Run Anywhere) : 빠르게 변화하는 시대에 발맞춰 살아남기 위해 나날이 발전 HTML(Hyper Text Mark-up Language) : 하이퍼 텍스트 마크업 언어 Hyper Text : 링크(Link) Mark-up Language : 태그(Tag)를 사용해서 문법을 표기하는 언어 프로그래밍 언어(x) / 표현 언어(o) : 브라우저 화면에 무언가 출력하는 기능만 있는 언어 웹사이트를 만들 때 쓰는 디자인 언어로 인터프리터 방식 컴퓨터 언어 : 컴파일러 vs 인터프리터 - 컴파일러 : Java > .java작업(.java 전체를 기계어로 번역(컴파일)) -> .class 실행 .java에 문제가 있어서 기계어(.class)로 번역이 불가능하게 된다. 즉, error가 발생하면 최종 산물인 .jar파일이나오지 않는다. - 인터프리터 : (HTML) .html작업 > 실행 > 한 줄씩 기계어로 번역해서 실행 잘못 만든 상황에서 이미 실행을 했으면 error는 안나옴 즉, 결과가 어떻게 될지 아무도 모름(문제 발생시 수정이 힘들 수 도 있다.)
